Dane pobierane są z API Gdańskich Wód.

Jeżeli coś nie działa to ich serwer dostał zadyszki i rzuca 502-ki. Z resztą cudem jest to, że to działa. Generalnie jest to proof of concept i ćwiczenie tworzenia aplikacji. Pod spodem zostało stworzone API które działa jako pośrednik dla frontu. Ukrywa ono złe kody błędu i różne dziwne praktyki serwowane przez pomiary.gdmel.pl/rest.

Niestety, aby pobrać dane z jednej stacji np. z tygodnia nasz pośrednik wykonuje 7 * 4 zapytań. Co oznacza, że Gdańskie Wody Sp. z o.o. otrzymuje około 2800 zapytań w chwili załadowania pierwszej strony. Dla frontu to tylko 94 zapytania, czyli tyle ile jest stacji.

Przepraszam.
